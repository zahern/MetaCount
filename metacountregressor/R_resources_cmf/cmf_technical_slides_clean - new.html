Crash Modification Factors

Hierarchical Count Modeling for Road Safety

Traffic Safety Analysis

2026-04-23

Motivation

When engineers face a design decision:

  • Should we widen lanes?
  • Should we reduce curves?
  • Should we change shoulder width?

We need to answer:

How much will crashes change if we implement a treatment?

That is exactly what a Crash Modification Factor (CMF) provides.

CMFs translate model coefficients into:

  • Percent crash change
  • Decision-ready safety impacts
  • Policy-relevant safety tradeoffs

What a CMF Actually Does

A CMF answers one question:

If we change a road feature from condition A to condition B how will predicted crashes change?

Example:

  • Lane width = 1.8 m → 2 m
  • Curve density = 2 → 3 per km
  • Shoulder width = .6 m → 1 m

The CMF tells us:

  • 0.85 → 15% crash reduction
  • 1.10 → 10% crash increase

What Are Crash Modification Factors?

CMF Definition:

\[\text{CMF}(a \rightarrow b) = \frac{\hat{N}_b}{\hat{N}_a}\]

where:

  • \(a\) = the current condition a.
  • \(b\) = after a treatment/change
  • \(\hat{N}_a\) = predicted crashes in condition \(a\)
  • \(\hat{N}_b\) = predicted crashes in condition \(b\)

Why CMF changes:

  • If the treatment lowers predicted crashes (\(\hat{N}_b < \hat{N}_a\)), then CMF < 1.
  • If the treatment raises predicted crashes (\(\hat{N}_b > \hat{N}_a\)), then CMF > 1.

**Example: Increase Width increases by 1 w

with \(\beta_{WIDTH}=-0.08394\)**

\[ \begin{aligned} \Delta \log(\hat N) &= -0.08394 \times 1 = -0.08394 \\ CMF &= \exp(-0.08394) = 0.9195 \\ \%\text{ change} &= 100\,(0.9195 - 1) = -8.05\% \end{aligned} \]

  • If the exponent is negative, then \(CMF < 1\) and crashes go down.
  • If the exponent is positive, then \(CMF > 1\) and crashes go up.

Conceptual Structure

Traditional Model

\[ \log(\hat N_i) = \text{Single additive equation} \]

That is

\[\hat{N}_i = E_i \times \exp(\beta_0 + \beta_1 X_{1i} + \beta_2 X_{2i} + \ldots)\]

Everything is estimated together:

  • Crash propensity
  • Traffic exposure
  • Traffic elasticity
  • Geometry effects

Hierarchical CMF Model

\[ \hat N_i = \underbrace{\exp(\text{baseline features})}_{\textbf{Inherent Risk}} \times \underbrace{AADT^{(\text{elasticity modified by geometry})}}_{\textbf{Traffic Response}} \]

Two distinct mechanisms:

  • Baseline block -> inherent crash propensity
  • Traffic-response block -> how crashes scale with traffic

This structural separation is the key advantage: AADT response is modeled explicitly.

Traditional models usually treat AADT primarily through exposure, not as a separate mechanism.

What the Exposure (Offset) Really Does

Traditional model with exposure:

\[ \hat N_i = E_i \times \exp(\beta_0 + \beta_1 X_{1i} + \dots) \]

If we assume AADT multiplied by segment length is the exposure:

\[ E_i = AADT_i \times LENGTH_i \]

Then:

\[ \log(\hat N_i) = \log(AADT_i) + \log(LENGTH_i) + \text{linear predictor} \]

Important

Including AADT in the offset:

  • Traffic elasticity remains fixed
  • Assumes proportional exposure
  • Makes it harder for geometry to directly modify traffic scaling

It assumes:

If traffic doubles, crashes double — regardless of geometry. Treatments are likely to cause changes in traffic

What the Hierarchical CMF Structure Allows

Baseline block:

\[ \hat N_i^{\text{baseline}} = \exp(\alpha_0 + \alpha_1 X_{1i} + \dots) \]

Traffic-response block:

\[ \hat N_i = \hat N_i^{\text{baseline}} \times AADT_i^{(\gamma_0 + \gamma_1 \text{WIDTH}_i + \gamma_2 \text{CURVES}_i)} \]

Now geometry can:

  • Affect inherent crash risk through the baseline channel
  • Independently modify traffic elasticity

Traffic scaling becomes geometry-dependent. For example, changes in WIDTH and CURVES can directly adjust AADT response.

The Problem: Traditional Models

Traditional negative binomial regression:

\[\hat{N}_i = E_i \times \exp(\beta_0 + \beta_1 X_{1i} + \beta_2 X_{2i} + \ldots)\]

Three Critical Questions

  1. What causes the coefficient?
    • Does CURVES increase crashes directly?
    • Or does it reflect traffic volume confounding (AADT)?
  2. How do we design interventions?
    • Should we prioritize geometry changes or traffic management?
    • Are both quantified?
  3. What if exposure assumptions are wrong?
    • Exposure effect should account for differences in road segmentation
    • But maybe longer roads have different safety dynamics

Solution: Hierarchical CMF Model

Two-Level Structure

Level 1 — Baseline Block (inherent crash propensity): \[\hat{N}_i^{\text{baseline}} = \exp(\alpha_0 + \alpha_{\text{URB}} \cdot \text{URB}_i + \alpha_{\text{ACCESS}} \cdot \text{ACCESS}_i + \cdots)\]

Level 2 — Traffic-Response Block (AADT elasticity): \[\hat{N}_i = \hat{N}_i^{\text{baseline}} \times \text{AADT}_i^{(\gamma_0 + \gamma_{\text{WIDTH}} \cdot \text{WIDTH}_i + \cdots)}\]

Key Advantage

  • Baseline features (URB, ACCESS, …) → What geometric factors directly explain the crash process
  • AADT features (WIDTH, CURVES, …) → How does the geometric factors affect the traffic volume sensitivity change?

Experiment Setup

Washington Rural Roads from 1990

  • Dataset: 275 road segments, 4-year crashes
  • Median AADT: 23,771 vehicles/day
  • Variables used: URB, ACCESS, GRADEBR, CURVES, WIDTH, AADT, LENGTH

Fitted Hierarchical CMF Model

Hierarchical CMF model

Structural form

\[ \log(\hat{N}_i) = \log(L_i) + \underbrace{\left(\alpha_0 + \boldsymbol{\alpha}^\top\mathbf{x}_i\right)}_{\text{baseline block}} + \underbrace{\left(\gamma_0 + \boldsymbol{\gamma}^\top\mathbf{z}_i\right)\log(AADT_i)}_{\text{traffic-response block}} \]

Fitted coefficients

\[ \begin{aligned} \log(\hat{N}_i) = {} & \log(L_i) \\ & + \underbrace{\left(-6.394 - 0.020\,URB + 0.141\,ACCESS + 0.097\,GRADEBR - 0.993\,CURVES - 1.947\,WIDTH\right)}_{\text{baseline block}} \\ & + \underbrace{\left(0.812 + 0.123\,CURVES + 0.175\,WIDTH\right)\,\log(AADT_i)}_{\text{traffic-response block}} \end{aligned} \]

How coefficients become CMFs Hierarchical

For a treatment that appears only in the baseline block:

\[ \begin{aligned} \log\!\left(\frac{\hat N_{after}}{\hat N_{before}}\right) &= \alpha_j\,\Delta x_j \\ CMF &= \exp\!\left(\alpha_j\,\Delta x_j\right) \end{aligned} \]

For a treatment that also changes the traffic-response block:

\[ \begin{aligned} \log\!\left(\frac{\hat N_{after}}{\hat N_{before}}\right) &= \left(\alpha_j + \gamma_j\log(AADT)\right)\,\Delta x_j \\ CMF(AADT) &= \exp\!\left[\left(\alpha_j + \gamma_j\log(AADT)\right)\,\Delta x_j\right] \end{aligned} \]

Baseline WIDTH -1.947 road width shifts inherent crash risk
Traffic-response \(\gamma_0\) (base elasticity) +0.812 how crashes scale with traffic in general
Traffic-response CURVES \(\times\) log(AADT) +0.123 curvy roads attract more crashes as traffic grows
Traffic-response WIDTH \(\times\) log(AADT) +0.175 wider roads attract more crashes as traffic grows

Key point: the CMF treatment effect can change through two channels: a baseline shift and a traffic-sensitivity shift.

What is changing?

  • Traditional: only the fixed WIDTH slope \(\beta_{WIDTH}\) changes the prediction ratio.
  • CMF: the WIDTH effect is recalculated at each AADT because both baseline and traffic-response matter.

CURVES Treatment: Traditional vs CMF

Intervention: reduce CURVES by \(-1\).

Traditional — fitted formula

\[ CMF = \exp\!\bigl(\beta_{CURVES}\cdot\Delta CURVES\bigr) = \exp(0.056 \times -1) = 0.946 \quad\Rightarrow\quad -5.4\% \]

(same regardless of AADT)

CMF — fitted formula

\[ \begin{aligned} CMF(AADT) &= \exp\!\Bigl[\bigl(\alpha_{CURVES} + \gamma_{CURVES}\,\log(AADT)\bigr)\times(-1)\Bigr] \\ &= \exp\!\Bigl[0.993 - 0.123\,\log(AADT)\Bigr] \end{aligned} \]

What is changing conceptually?

  • Traditional: reducing CURVES moves one pooled slope, so every corridor gets the same estimated reduction.
  • CMF: reducing CURVES changes both the baseline term and the traffic-scaling term, so higher AADT corridors get a larger estimated benefit.
AADT Level \(0.993 - 0.123\,\log(AADT)\) CMF % Change
10,000 \(0.993 - 0.123\times 9.21 = -0.140\) 0.869 -13.1%
23,771 (median) \(0.993 - 0.123\times 10.08 = -0.247\) 0.781 -22.0%
60,000 \(0.993 - 0.123\times 11.00 = -0.360\) 0.698 -30.3%

Interpretation: both models suggest fewer curves reduce crashes, but only CMF changes the size of that reduction with traffic level.

Reducing curves has a bigger impact on higher-traffic roads because it improves traffic flow and reduces conflict points, which is captured by the traffic-response block in the CMF model.

Traditional vs CMF: Practical Narrative

What traditional NB is doing:

  • One effect per treatment variable.
  • Easy to summarise, but it pools all segments into one treatment slope and cannot observe what the intended affect will result on traffic.

What hierarchical CMF is doing:

  • Baseline channel for general treatment effects.
  • AADT-response channel for traffic-conditioned treatment effects.
  • Better for deciding where a treatment should be prioritised.

Final Takeaway

CMF can be applied in two distinct ways:

  1. General treatments: baseline CMFs (traffic level not required).
  2. AADT-sensitive treatments: CMFs computed at chosen traffic levels.

That is the key modeling difference versus traditional NB regression:

  • Traditional usually reports one pooled treatment effect.
  • CMF separates baseline risk from traffic scaling, then lets treatment effects vary by traffic context where the model supports it.

\[ \text{Crash Risk} = \textbf{Baseline Propensity} \times \textbf{Traffic Scaling Behavior} \]
