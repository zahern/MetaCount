Methodology

Hierarchical CMF Count Model

The model specifies a two-level hierarchical log-linear count model:

\[ \log(\hat{\mu}_i) = \alpha + \underbrace{\beta_{\text{AADT}} \log(\text{AADT}_i)}_{\text{base AADT effect}} + \underbrace{\sum_{j \in U} \beta_j x_{ij}}_{\text{upper-level terms}} + \underbrace{\sum_{k \in L} \gamma_k x_{ik} \log(\text{AADT}_i)}_{\text{lower-level (AADT interaction) terms}} + \text{offset}_i \]

where \(\hat{\mu}_i\) is the expected crash count for segment \(i\).

Upper-level terms \(U\)

Additive effects on log-crash-rate, independent of traffic volume. Examples: road geometry, lane counts, pavement friction, climate.

Lower-level terms \(L\)

Modify the AADT elasticity — how crash risk scales with volume. Examples: curvature, friction, grade, precipitation.

\[\text{Local AADT elasticity}_i = \beta_{\text{AADT}} + \sum_{k \in L} \gamma_k x_{ik}\]

CMF for variable \(j\) at a given AADT: \(\text{CMF}(x_j) = \exp(\beta_j \cdot (x_j - x_j^{\text{ref}}))\) for upper terms, with AADT-varying response for lower terms.

Search Strategy

Random Search over Variable Subsets

Algorithm

  1. Seed with two baseline models (single-var and null)
  2. For each of 120 iterations:
    • Draw \(k_U \sim \text{Uniform}[0, 6]\) upper vars
    • Draw \(k_L \sim \text{Uniform}[0, 5]\) lower vars
    • Sample without replacement from candidate pools
    • Fit Poisson/NB model on training split (60%)
    • Score on validation split (20%) by Poisson deviance
    • Track AADT elasticity — enforce monotonicity constraint
  3. Select best monotonic model; fall back if none found
  4. Final refit on train + validation (80%)
  5. Report metrics on held-out test split (20%)

Scoring metrics

Metric Use
Val Poisson Deviance Primary selection criterion
BIC Complexity-penalized fit
Val RMSE Secondary diagnostic
AADT elasticity min Monotonicity guard

Why Poisson deviance?

Proper scoring rule for count data. Penalizes both over- and under-prediction relative to mean, and is consistent for mean estimation even under overdispersion.

Why enforce monotonicity?

Road safety theory requires more traffic → more crashes (positive elasticity). Models violating this are physically implausible.

CMF Interpretation

What Each CMF Represents

A Crash Modification Factor (CMF) answers: if this road characteristic changes, by how much does crash risk multiply?

\[\text{CMF} = \frac{\hat{\mu}(\text{modified})}{\hat{\mu}(\text{baseline})}\]

Upper-level CMF (AADT-independent)

Scaling factor is constant across traffic levels.

Example — CURVES: - More curves → higher CMF (CMF > 1 → more crashes) - Effect size fixed regardless of AADT

Lower-level CMF (AADT-varying)

Scaling factor changes with traffic volume.

Example — FRICTION × log(AADT): - At low AADT: friction effect modest - At high AADT: friction deficiency multiplied

This captures how the same road deficiency is more dangerous on a busy road.

Illustrative example: CURVES at three AADT levels

CURVES (z-score) Low AADT Median AADT High AADT
−1 SD (few curves) 0.78 0.82 0.87
0 (median) 1.00 1.00 1.00
+1 SD (many curves) 1.28 1.22 1.15

Values shown are illustrative; actual values depend on fitted coefficients.

The AADT-varying CMF is the core innovation of the hierarchical model: it captures effect modification between geometry and traffic volume.

Random Parameters

Capturing Unobserved Heterogeneity

Standard fixed-effects models assume all sites respond identically to the same covariate value. In practice, unobserved site characteristics cause the same geometry to have different effects across locations.

Random-parameters extension

\[\beta_{ij} = \bar{\beta}_j + \eta_{ij}, \quad \eta_{ij} \sim \mathcal{N}(0, \sigma_j^2)\]

  • Each segment \(i\) gets its own draw of \(\beta_j\)
  • Estimated via simulated maximum likelihood or Bayesian HMC
  • Uses ExperimentBuilder.fit_manual_model if available

Implementation in this script

  • Fixed terms: log(AADT), upper vars (standardised), lower interaction terms
  • Random (independent) terms: continuous upper vars
  • Correlated random terms: top-2 continuous lower vars (captures co-movement between e.g. curvature and friction effects)
  • Dispersion = 1 (NB2 parameterisation)

Benefits

Correct standard errors
Fixed-parameter SEs are too small when heterogeneity is ignored.
Segment-specific elasticities
Each site has its own AADT elasticity posterior.
Better out-of-sample calibration
Shrinkage regularises extreme coefficient estimates.

Output

  • random_params_summary.json written if ExperimentBuilder is installed
  • Contains full summary text of the mixed model

Note: requires optional experiment_package dependency. The script falls back gracefully if unavailable.

Key Findings

Why Hierarchical CMFs Matter

Methodological contributions

  • Unified framework for estimating both level and AADT-sensitivity of crash risk
  • Random search over variable subsets avoids specification bias from manual stepwise selection
  • Monotonicity constraint on AADT elasticity enforces physical plausibility
  • Train / validation / test split prevents over-fitting and gives honest performance estimates
  • Standardised coefficients enable direct comparison of effect magnitudes across variables

Practical benefits for safety analysts

  • Interactive CMF dashboard lets practitioners explore what-if scenarios without re-running models
  • AADT-varying CMFs distinguish high-volume from low-volume road risk profiles
  • Binary variable toggle buttons clearly communicate on/off safety treatments
  • Convergence plot diagnoses search quality and identifies if more iterations are needed

Conclusions

Why This Matters for Safety Planning

The hierarchical CMF framework provides:

  1. Transparent model structure — the two-level log-linear form is interpretable by practitioners and directly produces CMFs
  2. AADT-varying effects — safety treatments that have different effectiveness under light vs. heavy traffic can be correctly estimated and communicated
  3. Automated variable selection with explainable search history and convergence diagnostics
  4. Reproducible workflow — single Python script, CSV input, all outputs written to a dated results directory
  5. Interactive communication tool — the HTML dashboard makes model outputs accessible to non-statistician decision-makers

Limitations and future work

  • Random search does not guarantee global optimum; Bayesian optimisation or full enumeration could improve selection
  • Correlated random parameters require ExperimentBuilder (optional dependency)
  • Temporal dynamics (year effects, trend) not currently modelled
  • Spatial autocorrelation between nearby segments not explicitly addressed
